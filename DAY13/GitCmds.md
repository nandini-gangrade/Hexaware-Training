# GIT Commands

| Command                               | Description                                                                                              | Example Usage                                            |
|---------------------------------------|----------------------------------------------------------------------------------------------------------|----------------------------------------------------------|
| git init                              | Initializes a new Git repository in the current directory.                                               | git init                                                 |
| git clone [repository_url]            | Clones a remote repository from the specified URL to your local machine.                                 | git clone https://github.com/example/repository.git      |
| git add [file_name]                   | Adds changes in the specified file to the staging area.                                                  | git add file.txt                                         |
| git add .                             | Adds all changes in the current directory and its subdirectories to the staging area.                    | git add .                                                |
| git add --patch [file_name]           | Interactively selects changes in the specified file to add to the staging area.                          | git add --patch file.txt                                |
| git commit -m "[commit_message]"      | Commits the staged changes with a descriptive commit message.                                           | git commit -m "Add new feature"                         |
| git status                            | Displays the current status of the repository.                                                           | git status                                               |
| git status --short                    | Displays the current status of the repository in a concise, abbreviated format.                          | git status --short                                       |
| git log                               | Displays a list of commits in reverse chronological order.                                               | git log                                                  |
| git branch                            | Lists all branches in the repository and indicates the current branch with an asterisk (*).              | git branch                                               |
| git branch [branch_name]              | Creates a new branch with the specified name.                                                           | git branch new-feature                                   |
| git checkout [branch_name]            | Switches to the specified branch.                                                                        | git checkout main                                        |
| git merge [branch_name]               | Merges the changes from the specified branch into the current branch.                                    | git merge new-feature                                    |
| git pull                              | Fetches changes from the remote repository and merges them into the current branch.                      | git pull origin main                                     |
| git push                              | Pushes local commits to the remote repository.                                                           | git push origin main                                     |
| git remote -v                         | Lists all remote repositories associated with the current repository along with their URLs.             | git remote -v                                            |
| git diff                              | Shows the difference between the working directory and the staging area.                                 | git diff                                                 |
| git diff --staged                     | Shows the difference between the staging area and the last commit.                                       | git diff --staged                                        |
| git reset [file_name]                 | Unstages changes in the specified file, keeping them in the working directory.                           | git reset file.txt                                       |
| git reset --soft [commit_hash]        | Moves the HEAD to a specified commit, preserving staged and unstaged changes.                             | git reset --soft HEAD~1                                  |
| git reset --hard [commit_hash]        | Resets the working directory and staging area to match the specified commit.                             | git reset --hard 1234567                                 |
| git revert [commit_hash]              | Reverts the changes introduced by a specified commit, creating a new commit to record the reversal.      | git revert 1234567                                       |
| git cherry-pick [commit_hash]         | Applies the changes introduced by a specified commit onto the current branch.                             | git cherry-pick 1234567                                  |


| Flag                 | Description                                                                                                                                                                | Example Usage                                            |
|----------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------|
| -m, --message        | Specifies a commit message inline with the command.                                                                                                                         | git commit -m "Add new feature"                         |
| -p, --patch          | Interactively selects changes to be added to the staging area, allowing you to review each change (or "hunk") and choose whether to stage it.                              | git add -p file.txt                                     |
| -a, --all            | Adds all modified and deleted files to the staging area, including untracked files.                                                                                       | git add -A                                              |
| -v, --verbose        | Provides additional information or detailed output for the command being executed.                                                                                         | git status --verbose                                    |
| -1, -2, -3           | Specifies the number of lines context to show during a merge conflict resolution. `-1` shows the common base, `-2` shows the current branch changes, and `-3` shows the incoming changes. | git merge -1                                            |
| --no-verify          | Bypasses pre-commit and commit-msg hooks, allowing you to skip pre-commit checks and validations.                                                                          | git commit --no-verify                                  |
| --amend              | Modifies the last commit by combining staged changes with the previous commit and optionally updating the commit message.                                                 | git commit --amend                                      |
| --author [name]      | Overrides the default author name and email address for the current commit.                                                                                                | git commit --author "John Doe <john@example.com>"       |
| --date [date]        | Sets the author date for the current commit.                                                                                                                               | git commit --date "2022-05-25T10:00:00"                 |
| --dry-run            | Simulates the command without actually executing it, providing a preview of the changes that would occur.                                                                  | git push --dry-run                                      |
| --short              | Provides a concise, abbreviated output for commands such as `git status`, `git branch`, and `git log`.                                                                     | git status --short                                      |
| --color[=<when>]     | Enables or disables colorized output for Git commands.                                                                                                                      | git log --color=always                                  |
| --graph              | Displays commit history as a text-based graph, showing branch and merge history.                                                                                            | git log --graph                                         |
| --abbrev-commit      | Displays abbreviated commit hashes in Git output, reducing the length of commit IDs for brevity.                                                                           | git log --abbrev-commit                                 |
| --no-abbrev-commit   | Displays full commit hashes in Git output, showing the complete length of commit IDs.                                                                                       | git log --no-abbrev-commit                              |

> These highlighted search terms make it easier to identify the specific flags being discussed in each combination.
These flags provide additional functionality and customization options for various Git commands, allowing you to tailor your version control workflow to your specific needs and preferences.


| Combination          | Description                                                                                                                                                                                                                                                                                  | Example Usage                                          |
|----------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------|
| -p, --pickaxe <string> | Searches for changes that introduced or removed lines matching the specified string, displaying each change (or "hunk") and allowing you to choose whether to stage it. This is useful for finding specific changes using a "pickaxe" search. Combining with `-p` allows interactive selection of changes. | git log -p --pickaxe="/cool/"                            |
| -v, --verbose --short  | Provides detailed information about the status of the repository while maintaining a concise, abbreviated output. This can be useful when you need additional context for the changes displayed in a short format.                                                                              | git status -v --short                                  |
| -m, --message --author <name> | Specifies a commit message inline with the command and overrides the default author name and email address for the current commit. This allows you to quickly commit changes with a custom message and author without using separate commands.                                             | git commit -m "Fix issue" --author "John Doe"          |
| --color --graph          | Enables colorized output for Git commands while displaying the commit history as a text-based graph. This combination enhances readability and visual clarity when viewing the commit history in a graphical format.                                                                           | git log --color --graph                                |
| --abbrev-commit --no-abbrev-commit | Displays commit history with both abbreviated and full commit hashes in Git output. This combination allows you to quickly identify commits using shorter IDs while still having the option to view complete commit IDs when necessary.                                           | git log --abbrev-commit --no-abbrev-commit             |
| --date --short           | Sets the author date for the current commit and displays commit dates in a concise, abbreviated format. This combination provides control over commit timestamps while maintaining a compact output format for readability.                                                                        | git commit --date "2022-05-25T10:00:00" --short       |
| --dry-run --verbose      | Simulates Git commands without actually executing them and provides detailed information about the actions that would be performed if the commands were run. This combination allows you to preview changes and understand their implications before applying them.                    | git push --dry-run --verbose                           |
